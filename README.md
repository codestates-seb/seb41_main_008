## ⏰ Project Rules

### 1 Git Commit Convention 지키기
- 기본 형식 : [commit type]: [commit message]
- commit type
    - feat : 기능 구현
    - fix : 버그 수정
    - docs : 문서 작업
    - refactor : 리팩토링
    - test : 테스트
    - chore : 기타 작업

### 2. Branch 생성 규칙
- 기본 형식 : [type]/[feature]
- 기능 구현이 완료되면 Backend/Frontend Branch PR 남기기
- PR 후 Merge된 브랜치는 삭제하기
